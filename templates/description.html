Basic outline of the dataset :

  • The dataset is a large scale dataset comprising of over 6 million (6 * 106) records and 10 fields that contain transaction data and whether those transaction were detected as fraudulent or not.
  • It is a highly heterogeneous dataset which has fields of variegated datatypes. For instance, fields - type, nameOrigin and nameDest are Strings whereas amount and balance based fields are of float types. There are two binary fields as well, isFraud and isFlagged fraud.
  • isFlagged and isFraud have binary values, they can be treated as boolean (true/false) or integers (0/1) depending upon nature of data analysis. isFraud is actual detection of a fraudulent transaction whereas isFlaggedFraud is the prediction made by the current algorithm whether fraudulent or not.
  • It is the isFlaggedFraud field that will be the prediction class, meaning that the prediction for fraudulence will made into it this field based on the values of the other fields, as defined in the feature set.
  • A post processing analysis reveals that the mean amount and the median amount vastly differ. This shows the intrinsic skewness in dataset pertaining to the transaction "amount" where there are lot of extreme values. And this skewness most likely permeates to the records as well. So, an exact prediction is difficult.

Issues with the dataset :

  • A major issue with this dataset is that out of over 6 million records, there are only 2 instances which are fraudulent. With such a less number of isFraud data, a robust algorithm cannot be build.
  • Even out of those two samples, although they are fraudulent, the current algorithm does not predicted them as fraudulent in the isFlaggedFraud field. Which means there is virtually no True Positive instance for a fraudulent transaction being classified as it is supposed to be. This may negatively affect the precision and overall accuracy of any ML-algorithm.
  • Instead of using 0/1 for binary values, a better alternative could have been using true/false, as boolean values enable better binary classification and give a better notation from a predictive analysis point-of-view.
