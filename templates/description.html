Basic outline of the dataset :

  • The dataset is a large scale dataset comprising of over 6 million (606) records and 10 fields that contain transaction data and whether those transaction were detected as fraudulent or not.
  • It is a highly heterogeneous dataset which fields being of variegated datatypes. For instance fields - type, nameOrigin and nameDest are Strings whereas amount and balance based fields are of float types. There are two binary fields as well, isFraud and isFlagged fraud.
  • isFlagged and isFraud have binary values, they can be treated as boolean(true/false) or even integers (1/0) depending upon nature of data analysis. isFraud is actual detection of fraudulent whereas isFlaggedFraud is that the current algorithm has flagged it as fraudulent or not.
  • It is the isFlaggedFraud field that will acts as the prediction class, meaning that the prediction will made into it this field based on the values of the other fields.
  • A post processing analysis reveals that the mean amount and the median amount vastly differ. This shows the intrinsic skewness in dataset pertaining to the transaction "amount" where there are lot of extreme values. And this skewness most likely permeates to the records as well. So, an exact prediction is difficult.

Issues with the dataset :

  • A major issue with this dataset is that out over 1 million records, there are only 2 instances which are fraudulent. With such less number of isFraud data, a robust algorithm cannot be build.
  • Even out of those two samples, although they are fraudulent, the current algorithm does not flag them as fraudulent. Which means there is virtually no True Positive instance for a fraudulent transaction being classified as it is supposed. This may negatively affect the precision and overall accuracy of any ML-algorithm.
  • Instead of using 1/0 for binary values, a better alternative could been using true/false as boolean values enable better binary classification and given a better notation from a predictive analysis point-of-view.
